#기말고사
1. 브랜치와 원격 저장소 심화 (15강 ~ 18강)
기본적인 브랜치 생성과 원격 저장소 연결을 넘어, 실제 협업 흐름을 다룹니다.

브랜치 관리:

git branch -d [이름]: 다 쓴 브랜치 삭제 (병합되지 않은 브랜치는 삭제 거부됨).

git switch [이름]: 브랜치 이동 (Checkout과 동일).

원격 연동 흐름:

Clone: git clone [주소] (최초 복제).

Push: git push origin [브랜치명] (업로드).

Pull: git pull origin [브랜치명] (다운로드 + 병합).

인증: GitHub 비밀번호 대신 PAT (Personal Access Token) 사용 필수.

2. 오픈소스와 임시 저장 (Stash) (19강 ~ 21강)
작업 도중 급하게 다른 브랜치로 가야 할 때 유용한 기능과 오픈소스 개념입니다.

1. 오픈소스 소프트웨어 (OSS):

개념: 소스 코드가 공개되어 누구나 자유롭게 확인, 수정, 배포할 수 있는 소프트웨어.

주요 라이선스:

MIT: 가장 제약이 적음 (저작권 고지만 하면 됨).

Apache 2.0: 특허권 포함, 수정 시 아파치 라이선스 명시 필요 (예: 안드로이드).

2. 임시 저장 (Stash):

작업 중인 파일(Commit 전)을 잠시 보관함에 넣어두고, 작업 디렉토리를 깨끗하게 비움.

git stash (또는 save): 현재 작업 내용 임시 저장.

git stash list: 저장된 목록 확인 (stash@{0}, stash@{1}...).

git stash pop: 가장 최근 저장 내용을 꺼내와 적용하고 목록에서 삭제.

git stash apply: 내용을 적용하되 목록에 그대로 유지.

3. 고급 병합 (Merge) 기법 (22강 ~ 23강)
브랜치를 합치는 3가지 방식을 구분하는 것이 중요합니다.

병합 방식  --  설명  --  특징

Fast-Forward: 브랜치가 갈라지지 않고 일직선일 때 - 별도의 커밋 없이 포인터만 앞으로 이동.

3-Way Merge: 브랜치가 갈라져서 내용이 다를 때 - 두 브랜치를 합치는 새로운 '병합 커밋' 생성.

Squash Merge: git merge --squash 코드사용시 - 병합할 브랜치의 모든 커밋을 하나로 압축하여 병합. 지저분한 이력을 정리할 때 유용.


4. 충돌 (Conflict) 해결 (24강)
Git이 자동으로 합치지 못하는 상황(같은 라인을 서로 다르게 수정)에서의 대처법입니다.

충돌 발생: Merge 시 CONFLICT 에러 메시지 출력.

코드 수정: 파일 내의 <<<<<<<, =======, >>>>>>> 표시를 찾아 원하는 코드로 수정.

해결 완료: git add [파일] -> git commit (별도 메시지 없이 저장하면 자동 생성됨).

5. 이력 관리: Rebase와 수정 (25강 ~ 27강)
이력을 깔끔하게 만들거나, 방금 한 실수를 고치는 방법입니다.

1. Rebase (재배치):

git rebase [브랜치]: 내 브랜치의 시작점(Base)을 최신 위치로 옮겨 붙임.

특징: 이력이 한 줄(Linear)로 깔끔해지지만, 이미 원격에 올린 커밋은 Rebase 금지 (동료들의 이력이 꼬임).

2. 커밋 수정:

git commit --amend: 방금 한 최신 커밋의 메시지나 내용을 수정 (덮어쓰기).

git rebase -i (Interactive): 과거의 여러 커밋을 합치거나(squash), 메시지 변경(reword), 삭제(drop)하는 강력한 도구.

3. VS Code 활용:

Git Graph: 복잡한 터미널 명령 대신 GUI로 이력 시각화, 우클릭으로 Rebase/Merge 수행 가능.

6. 버전 되돌리기: Reset vs Revert (28강 ~ 30강)
가장 헷갈리기 쉽고 시험에 자주 나오는 되돌리기의 두 가지 방식입니다.

1. Reset (과거로 이동):

특정 시점으로 시간을 되돌림. 이후의 이력은 사라짐.

git reset [옵션] [커밋ID]

--soft: 커밋만 취소 (파일은 스테이징 영역에 보존).

--mixed (기본값): 커밋+스테이징 취소 (파일은 작업 디렉토리에 보존).

--hard: 완전 삭제. 커밋+스테이징+작업내용 모두 날림 (주의!).

2. Revert (취소 커밋 생성):

git revert [커밋ID]: 특정 커밋의 내용을 정반대로 수행하는 새로운 커밋을 만듦.

특징: 기존 이력을 삭제하지 않고 '취소했다'는 기록을 남김. 이미 원격에 Push한 내용을 되돌릴 때는 반드시 Revert를 사용해야 함.
